[concept.txt] - LogAnalyzer GUI Tool

Features:
1. Запоминать текущую конфигурацию автоматически;
  - файлсеты
2. Панель файлов:
  - fileset - набор файлов для анализа
  - каждый файлсет имеет имя (по-умолчанию генерируется по текущей дате + порядковый номер: 2017-10-09_fs1)
  - fileset может быть:
  	- отредактирован руками (файлы добавлять по одиночке)
    - сканировать файлы из каталога (с рекурсией и без рекурсии, по маске, без маски)
    - можно включать/выключать отдельные файлы для анализа логов
    - задавать кодировку для каждого файла
    - задавать паттерн для парсинга даты-времени для каждого файла
  	- drag-n-drop файла или каталога
3. Панель фильтров:
  - строка выражения фильтра
  X паттерны для определения логических элементов лога (дата, время, поток, уровень логирования, класс)
  - пропуск первых элементов (skip)
  - ограничение вывода (limit)
  - from, till, check
4. Главное окно:
  - имеет имя (по умолчанию генерируется автоматически), можно задавать
  - можно сохранять результаты поиска
  - в качестве источника может быть файлсет или другое окно с результатами поиска


Сценарий работы:
1. Создаем новый fileset: Menu->File->New Fileset или Menu->View->Fileset - открывает окно fileset, где будет доступнка кнопка New Fileset;
2. На окне fileset: - add file, add dir, [ ] recursive, mask: _____; remove, select all, deselect all
3. На тулбаре выбираем текущий файлсет (по-умолчанию выбран новый)
4. На панели с фильтрами задаем запрос
5. На тулбаре нажимаем PLAY:
  - открывается окно search results в которое выводятся строки, удовлетворяющие условию;
  - фон для строк для разных файлов чередуется;
  - вывод возможен в ListView или в text editor с шрифтом monospace;
6. Вседа доступнка кнопка STOP
7. Результаты выборки можно сохранить в новый файл, с именем: 20171009_sr1.log
